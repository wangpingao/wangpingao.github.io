Hexo博客搭建入门教程

发表于 2018-05-07

前言：博主目前大三，Java菜鸟级选手，搭建博客主要为了记录自己学习的心得体会，也欢迎各路大神指教。

第一篇博客就记录一下使用Hexo搭建的完整过程。

个人博客地址：https://wangpingao.github.io/

大致流程：

  1. 搭建Node.js环境
  2. 搭建Git环境
  3. Github配置
  4. 安装配置Hexo
  5. 关联Hexo与Github Pages

搭建Node.js环境

为什么搭建Node.js环境？-因为Hexo博客系统是基于Node.js编写的

Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行环境，可以在非浏览器环境下，解释运行 JS 代码。

在Node.js官网：https://nodejs.org/en/ 下载安装包 v8.11.1 LTS

保持默认设置，一路Next。

打开命令提示符，输入 node -v、npm -v ,出现版本号说明安装成功。第一步完成。

搭建Git环境

为什么搭建Git环境？-因为需要把本地的网页和文章等提交到GitHub上。

Git 是一款免费、开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。

在Git官网：https://git-scm.com/ 下载安装包： Git-2.13.0-64-bit.exe

一路Next安装，安装成功。

打开 Git Bash ，输入 git --version ，出现版本号说明Git环境配置成功，第二步完成。

GitHub配置

GitHub 是一个代码托管平台，因为只支持 Git 作为唯一的版本库格式进行托管，故名 GitHub。

Github地址：https://github.com/

创建仓库： yourname.github.io

注意：yourname 必须是你的用户名

因为博主实现已经创建仓库，所以图片中提示已存在。

访问 yourname.github.io，如果可以正常访问，那么 Github 配置成功。

到此搭建 Hexo 博客的相关环境配置已经完成，下面开始讲解 Hexo 的相关操作。

安装配置Hexo

Hexo 是一个快速、简洁且高效的博客框架，使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

Hexo官网： https://hexo.io/zh-cn/

使用npm安装Hexo：在命令行输入 npm install hexo-cli -g
